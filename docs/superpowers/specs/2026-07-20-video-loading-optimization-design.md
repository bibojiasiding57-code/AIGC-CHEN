# AIGC-CHEN 视频加载优化设计

## 目标

在保持作品卡片布局、悬停预览、单击播放按钮、双击弹窗和移动端行为不变的前提下，提高 GitHub LFS 视频在跨境网络中的感知加载速度、缓冲反馈和失败恢复能力。

## 采用方案

采用按需预热策略，而不是同时预加载 14 个视频：

- 卡片初始使用 `preload="metadata"`，避免首屏并发下载全部视频。
- 卡片进入视口附近后，将视频提升为可预取状态并调用 `load()`。
- 指针或键盘焦点进入卡片时立即请求播放；离开时暂停并按现有行为归零。
- 同一时刻仅由用户正在交互的卡片主动播放，避免带宽竞争。
- 大尺寸弹窗保持 `autoPlay`，并使用更积极的 `preload="auto"`，优先保障当前选中作品。

## UI 与状态

新增不改变布局尺寸的覆盖层：

- `loadstart`、`waiting`、`stalled` 时显示低调的圆形加载指示和“视频缓冲中”。
- `canplay`、`playing` 时隐藏加载层。
- `error` 时继续使用现有媒体不可用回退，不覆盖现有播放按钮的点击层级。
- 加载覆盖层固定 `pointer-events: none`，不影响卡片、播放按钮和双击事件。

## 组件边界

- `MediaCard.jsx`：负责视口预热、悬停预览和卡片级缓冲状态。
- `WorksVideoDialog.jsx`：负责弹窗播放器的积极预载和缓冲状态。
- `styles.css`：只新增加载层样式，不修改卡片网格、尺寸、圆角或按钮位置。

## 性能与兼容性

- 使用单个卡片级 `IntersectionObserver`，在组件卸载时断开。
- 不支持 `IntersectionObserver` 时保持 `metadata` 预载，悬停仍可正常播放。
- 尊重现有 `muted`、`playsInline` 和循环设置；不新增第三方依赖。
- GitHub LFS 仍是网络上限，代码优化改善调度与反馈，但不能消除大陆跨境线路波动。

## 验证

- 测试视口进入后调用 `load()`。
- 测试缓冲事件显示 Loading、`canplay/playing` 隐藏 Loading。
- 测试覆盖层不阻挡点击。
- 测试弹窗使用 `preload="auto"` 并显示缓冲反馈。
- 运行全部 Vitest 测试和 Vite 生产构建。
