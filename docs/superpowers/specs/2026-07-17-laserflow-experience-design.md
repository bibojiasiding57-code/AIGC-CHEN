# AIGC-CHEN 互动体验区 LaserFlow 设计说明

## 目标

在现有“互动体验”区域中加入 React Bits `LaserFlow` 光束，并实现与示例一致的局部明度揭示：图片和视频默认压暗，鼠标或触摸位置周围以柔边圆形区域恢复媒体原本明度。效果只作用于互动体验区，不改变角色介绍、作品案例或联系方式。

## 已确认范围

- PC 端使用完整 LaserFlow WebGL 光束。
- 手机端和“减少动态效果”环境不运行 WebGL，改用轻量 CSS 暖光与触摸揭示。
- 保留现有两张图片、一段视频、标题、说明和“沉浸 / 柔光 / 脉冲”按钮。
- 不复制图片或视频，避免视频重复解码。
- 一次性加入 `three` 依赖；不搜索 pnpm 依赖，不进行重复联网校验。

## 视觉层级

互动体验区从底到顶分为五层：

1. 现有深色或情绪模式背景。
2. 暖橙金色 LaserFlow 光束层。
3. 原始媒体层，包括两张图片和一段视频。
4. 每个媒体独立的半透明深色遮罩层。
5. 标题、说明和情绪按钮内容层。

光束颜色使用 `#F6A04D`，与网站主色一致。光束作为氛围背景，不覆盖文字、按钮或改变媒体构图。

## 明度揭示

- 每个媒体由独立 `RevealMedia` 包装层承载，包装层包含原媒体与一个不可交互的深色遮罩。
- 遮罩默认使用约 55% 的深色透明度，使媒体明显压暗但仍能辨认内容。
- 光标中心的完全揭示区域约为 120px 半径，向外以约 120px 柔边过渡，总影响半径约 240px。
- 揭示只移除深色遮罩，因此恢复的是媒体明度；现有情绪模式的色彩滤镜继续生效。
- 鼠标离开互动区域后，遮罩在约 220ms 内平滑恢复。
- 触摸设备在按下或拖动时显示揭示区，松开后平滑淡出。

## 组件边界

### `LaserFlow`

- 文件：`src/components/LaserFlow.jsx` 与 `src/components/LaserFlow.css`。
- 以用户提供的 React Bits JavaScript + CSS 源码为基础。
- 接收互动区域引用作为指针目标，使光束与外层区域的指针坐标同步。
- 负责 WebGL 创建、尺寸变化、视口可见性、动画暂停、动态 DPR 与资源释放。
- 暖色调参数采用克制配置：降低雾强度、光丝密度与运动速度，避免抢夺媒体焦点。

### `RevealMedia`

- 文件：`src/components/RevealMedia.jsx`。
- 统一承载图片或视频、定位类名、遮罩和可访问性属性。
- 只负责显示与遮罩，不管理全局情绪状态或 LaserFlow 生命周期。
- 使用明确的布局修饰类替换依赖 `:first-child` / `:nth-child` 的媒体定位，避免加入包装层后选择器失效。

### `App`

- 保留互动区域引用和情绪状态。
- 指针移动通过 `requestAnimationFrame` 节流。
- 每帧最多为三个 `RevealMedia` 计算一次局部坐标，并写入 `--reveal-x`、`--reveal-y` 和激活状态。
- 指针离开、触摸结束、页面失焦时统一清理揭示状态。

## 响应式与性能

- 仅在宽度大于 `720px` 且 `prefers-reduced-motion: no-preference` 时创建 LaserFlow。
- PC 渲染 DPR 上限设为 `1.5`，同时保留组件根据帧率自动降低 DPR 的机制。
- LaserFlow 离开视口或页面进入后台时暂停动画。
- 手机端保留现有 CSS 指针光晕，并以同一套遮罩坐标实现触摸揭示。
- 所有光效层使用 `pointer-events: none`，不阻挡情绪按钮或页面滚动。

## 降级与错误处理

- WebGL 不可用、上下文创建失败或上下文丢失时，不抛出影响页面的错误；LaserFlow 隐藏并保留 CSS 暖光揭示。
- 图片或视频加载状态不依赖 LaserFlow，光效失败不会影响媒体浏览或按钮切换。
- 组件卸载时取消动画帧、断开观察器、移除事件监听并释放 geometry、material、renderer 与 WebGL 上下文。
- 手机端、减少动态效果模式和 WebGL 降级均保持相同的明度揭示语义。

## 依赖策略

- 在现有 React + Vite 项目中一次性加入 `three`，并记录到项目依赖清单。
- 不执行 pnpm 依赖搜索，不循环检查网络，不引入额外动画库。
- 若一次安装因外部网络或权限失败，停止并报告，不进行重复联网重试。

## 测试与验收

### 自动化测试

- `RevealMedia` 正确渲染媒体和遮罩。
- 指针移动后，三个媒体包装层收到对应的局部揭示坐标。
- 鼠标离开、触摸结束或页面失焦后，揭示状态清理。
- 手机宽度和减少动态效果环境不创建 LaserFlow WebGL 层。
- WebGL 初始化失败时仍保留互动内容和 CSS 揭示。
- 现有情绪切换与 `aria-pressed` 测试继续通过。
- 全量测试和 Vite 生产构建通过。

### 浏览器验收

- PC 端可见暖橙金色 LaserFlow，且光束不遮挡文字和按钮。
- 媒体默认明显压暗，光标附近自然恢复明度，圆形边缘无硬切。
- 图片与视频都能被揭示，视频持续正常播放且只有一个实例。
- 三种情绪模式切换正常，揭示效果只改变明度。
- 手机端滚动顺畅，触摸揭示有效，无持续 WebGL 动画。
- 浏览器控制台没有运行时错误或 WebGL 资源警告。

## 非目标

- 不修改互动体验区以外的页面。
- 不重排媒体构图，不替换现有图片和视频。
- 不增加声音、粒子点击反馈或新的交互按钮。
- 不复制媒体层，不为不同情绪模式创建额外视频实例。
