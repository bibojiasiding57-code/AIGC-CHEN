# AIGC-CHEN 个人作品集设计说明

## 目标

为 AI 设计师 **AIGC-CHEN** 制作一个单页 React + Vite 作品集。页面以用户提供的参考图 `D:\下载合集\generated_b05614a7-2982-4da1-b5b9-767c9bae9420_mzk-k0-lki6.png` 为视觉源，保留暖橙画布、大圆角影像、白色粗体标题、青蓝胶囊导航和奶油色作品卡，同时接入 `D:\做网站用的` 中的真实视频与海报素材。

## 视觉系统

- 页面底色：暖橙 `#F6A04D`，辅以深橙 `#EB7E2F`、奶油白 `#FFF6EA`、墨色 `#252A32`、青蓝 `#397782`。
- 最大版心：`1700px`；桌面左右留白 `clamp(18px, 3vw, 48px)`。
- Hero：大圆角视频容器，桌面高度约 `min(82svh, 860px)`；用深蓝半透明遮罩保证标题可读。
- 导航：Hero 下方居中胶囊组，四个锚点为“角色介绍 / 作品案例 / 互动体验 / 联系方式”。
- 字体：优先 `Arial`, `Helvetica Neue`, `PingFang SC`, `Microsoft YaHei`, sans-serif；标题使用 800–900 字重，正文 400–600。
- 圆角：Hero `30px`；内容卡 `26px`；按钮 `999px`。
- 动效：滚动出现、案例悬停、光标追随的暖色光晕；尊重 `prefers-reduced-motion`。

## 页面结构

1. **Hero**：循环静音视频、品牌标签、身份说明、主标题 “AIGC-CHEN · AI IMAGINATION”、查看作品按钮和滚动提示。
2. **角色介绍**：中英文简介、能力标签、项目数量与工作方向。
3. **作品案例**：视频与海报混排；首屏三列卡片对应 AI 影像、数字艺术、品牌视觉，后续网格继续展示案例。
4. **互动体验**：光标驱动的视觉舞台，提供三个情绪按钮切换界面色调和文案。
5. **联系方式**：电话与邮箱；复制邮箱按钮提供成功反馈。

## 组件边界

- `src/data/portfolio.js`：导航、能力、案例、联系方式等纯数据。
- `src/utils/ui.js`：平滑滚动目标解析、邮箱复制状态文案等纯函数。
- `src/components/MediaCard.jsx`：图片/视频案例卡，负责媒体类型渲染。
- `src/components/SectionHeading.jsx`：统一章节标题结构。
- `src/App.jsx`：页面编排、情绪切换与复制交互。
- `src/styles.css`：设计令牌、布局、响应式和动效。

## 交互与数据流

- 导航点击通过锚点定位到对应章节；键盘用户可正常聚焦和触发。
- 互动体验的情绪按钮只更新本地 React 状态，并通过 `data-mood` 驱动样式。
- 邮箱复制使用 Clipboard API；不可用时选中文本并显示手动复制提示。
- 视频均设置 `muted`, `loop`, `playsInline`；非 Hero 视频使用 `preload="metadata"` 和懒加载容器。

## 性能与响应式

- 首版优先使用体积较小的视频作为 Hero；大文件只作为案例链接或延迟加载。
- 图片放入 `public/media`，使用 `loading="lazy"` 与 `object-fit: cover`。
- `1100px` 以下两列，`720px` 以下单列；移动端收紧标题字号并允许胶囊导航横向换行。
- 避免持续高开销滤镜；玻璃模糊仅用于小面积导航和按钮。

## 错误处理

- 媒体加载失败时保留卡片标题和分类，并显示纯色回退面。
- Clipboard API 失败时显示“请手动复制邮箱”。
- 页面不依赖后端、账户或外部数据，首版无网络数据失败状态。

## 测试与验收

- Vitest 覆盖纯函数和核心页面行为：导航章节、案例数量、情绪切换、联系方式渲染。
- `npm run build` 必须成功。
- 在桌面 `1440 × 1024` 与移动端 `390 × 844` 检查无横向溢出、标题不裁切、导航和互动可用。
- 设计 QA 对比参考图和实现截图；检查字体、布局节奏、色彩、媒体裁切和文案，最终 `design-qa.md` 必须为 `final result: passed`。
