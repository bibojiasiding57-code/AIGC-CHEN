# AIGC-CHEN 首屏 SplitText 动效设计说明

## 目标

在现有首屏主标题 `AIGC-CHEN·WELCOME` 上集成用户指定的 React Bits `SplitText` 字符入场动效。标题在页面首次进入或刷新时逐字从下方淡入，完成后保持静止；用户滚动离开再返回时不重复播放。

本次改动只作用于首屏 `h1#hero-title`，不改变标题文案、首屏构图、导航、视频、其他区块或 LaserFlow 互动体验的既定设计。

## 已确认方案

- 采用方案 A：使用用户提供的 React Bits JavaScript + CSS 版本，并使用 `gsap`、`@gsap/react` 与 GSAP `SplitText` / `ScrollTrigger`。
- 动画单位为字符，方向为自下向上并淡入。
- 首次进入或刷新时只播放一次，不在再次滚入视口时重播。
- 保留现有标题的大字号、紧凑字距、单行排版、左对齐和 `h1` 语义。
- 动效与 LaserFlow 一并实施，但两个组件互不依赖，任一动效降级都不影响另一部分或页面内容。

## 组件边界

新增 `src/components/SplitText.jsx`，以用户提供的源码为基础，保留以下职责：

- 等待字体就绪后拆分标题并创建动画。
- 使用 GSAP `SplitText` 生成字符目标，使用 `ScrollTrigger` 控制进入视口时触发。
- 通过内部完成标记与 `once: true` 保证单次播放。
- 在卸载或重新初始化时撤销拆分、停止关联触发器并释放引用。
- 支持 `tag`、`text`、`className`、动画参数、对齐方式和完成回调。

为适配现有页面，组件额外支持并透传标准元素属性，包括 `id`、`aria-label` 和其他安全的 HTML 属性。该扩展不改变用户提供组件的默认行为。

拆分完成后，组件为文本内容是 `·` 的字符包装添加 `split-char--separator` 类。这样可以精确保留当前圆点两侧的留白，又不把标题专属样式写死到通用组件结构中。

`App.jsx` 只负责用 `SplitText` 替换当前标题标签并传入已确认参数，不把首屏其他状态或互动逻辑移入组件。

## 标题配置

首屏使用以下固定配置：

```jsx
<SplitText
  id="hero-title"
  aria-label="AIGC-CHEN · WELCOME"
  tag="h1"
  text="AIGC-CHEN·WELCOME"
  splitType="chars"
  delay={38}
  duration={0.85}
  ease="power3.out"
  from={{ opacity: 0, y: "0.42em" }}
  to={{ opacity: 1, y: 0 }}
  threshold={0.1}
  rootMargin="-100px"
  textAlign="left"
/>
```

`0.42em` 使用相对字号位移，使桌面与手机端的运动距离随标题大小同步缩放。完整动画约在标题出现后的短时间内结束，节奏有层次但不拖慢首屏阅读。

## 视觉与排版

- 继续沿用现有 `.hero h1` 的字号、字重、行高、字距、颜色与大写风格。
- `SplitText` 根元素保持全宽块级标题和左对齐，不引入居中或额外外边距。
- 标题在 PC 端保持单行；现有响应式字号继续控制手机端宽度，避免字符拆分改变换行结果。
- 将现有 `.hero h1 span` 间距规则收窄为 `.hero__title .split-char--separator`，只给 `·` 保留当前 `0 0.05em` 外边距。其他字符包装元素只承担动画，不改变字形或字符间距。
- 动画期间隐藏标题自身的溢出内容，避免下方起始位置出现多余字符；不得裁切最终字形。
- 无 JavaScript、降级或动画失败时仍显示完整静态标题。

## 触发、生命周期与容错

1. 组件挂载后等待 `document.fonts.ready`；若浏览器没有 Font Loading API，则直接继续初始化。
2. 标题进入设定阈值时播放一次。动画完成后记录完成状态，不因组件普通重渲染或再次滚入视口而重播。
3. 依赖参数变化导致重新初始化时，先撤销旧的拆分与触发器，避免重复包裹字符。
4. 组件卸载时只清理属于当前标题的动画与触发器，不清除页面中其他 GSAP 实例。
5. 如果 GSAP、SplitText 或初始化流程在运行时失败，恢复或保留完整静态标题，并避免错误阻断页面渲染。

## 减少动态效果与可访问性

- 当 `prefers-reduced-motion: reduce` 生效时，不拆分、不播放 GSAP 动画，直接渲染完整静态 `h1`。
- 标题继续使用 `h1#hero-title`，首屏区块的 `aria-labelledby="hero-title"` 关系保持有效。
- 根标题提供 `aria-label="AIGC-CHEN · WELCOME"`，辅助技术读取一条连贯文本，不逐个朗读视觉字符包装。
- 动画不接管焦点、不新增可交互元素，也不阻挡标题附近的链接或滚动操作。

## 依赖策略

- 与 LaserFlow 所需依赖合并为一次安装：`three`、`gsap`、`@gsap/react`。
- 不搜索 pnpm 依赖，不循环检测网络，不执行重复联网安装或校验。
- 若这一次安装因网络或权限失败，停止并报告，不自动重试。
- 不额外引入其他文字动画库。

## 测试与验收

### 自动化测试

- 首屏仍渲染唯一的 `h1#hero-title`，可访问名称为 `AIGC-CHEN · WELCOME`。
- 正常动态环境下使用确认的字符拆分参数创建动画。
- `prefers-reduced-motion: reduce` 时显示静态标题且不初始化 GSAP 拆分。
- 动画完成或再次滚入视口时不重复播放。
- 组件卸载时清理自身实例，不影响页面其他动画。
- GSAP 初始化异常时保留可读的静态标题。
- 现有全量测试与 Vite 生产构建继续通过。

### 浏览器验收

- 刷新首页后，标题逐字由下方自然淡入，字序、间距和圆点位置正确。
- 标题结束后清晰稳定；滚动离开再返回不重播。
- PC 端标题保持现有单行构图，不溢出 Hero 卡片。
- 手机端位移与字号同步缩小，页面滚动顺畅，无横向滚动条。
- 开启“减少动态效果”后标题立即完整可见。
- 浏览器控制台没有重复初始化、未清理触发器或运行时错误。

## 非目标

- 不改变 `AIGC-CHEN·WELCOME` 文案、字形、颜色或整体首屏布局。
- 不增加循环播放、悬停重播、滚动反向播放或逐词模式。
- 不把 SplitText 与 LaserFlow 指针或亮度揭示逻辑绑定。
- 不处理站点发布、域名、后端或内容管理功能。
