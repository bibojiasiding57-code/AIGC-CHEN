# AIGC-CHEN 互动体验视频与文字布局精修设计

## 目标

在保持互动体验舞台整体结构、情绪切换和右侧两张杂志风卡片完全不变的前提下，将左侧视频替换为“测试成功”素材，并缩小、下移左侧文字区，使视频主体获得更多可见空间。桌面端与 360px/390px 移动端均不得出现文字重叠、裁切或页面横向溢出。

## 素材绑定

- 左侧 `RevealMedia` 视频从 `/media/cloud-fantasy.mp4` 替换为已通过文件名匹配并收录的 `/media/works/test-success.mp4`。
- 视频可访问名称更新为“测试成功动态影像”。
- 移除旧的 `/media/fantasy-character.jpg` 海报，避免视频加载前显示与新素材无关的静态画面。
- 保留 `muted`、`loop`、`playsInline`、`autoPlay` 和 `preload="metadata"` 行为。

## 右侧冻结边界

- 不修改 `.experience-media--fashion` 与 `.experience-media--cyber` 对应的 JSX、素材路径、尺寸、位置、旋转、层级或 RevealMedia 行为。
- 不修改右侧情绪切换器的位置和交互。
- 不调整互动舞台整体高度、背景、LaserFlow 或右侧卡片所在的媒体容器结构。

## 桌面端文字布局

- `.experience-stage__copy` 保持绝对定位、`left: 7%` 与现有层级，`bottom` 从 `11%` 调整为 `6%`。
- 大标题保持两行语义与现有换行，字号从 `clamp(40px, 5.4vw, 88px)` 调整为 `clamp(36px, 4.7vw, 76px)`。
- 保留标签、大标题、中文说明的三层顺序和现有主色；只在需要时微调行间距以避免压缩后的视觉松散。
- 中文说明继续位于标题下方，不覆盖情绪切换器或舞台底部边界。

## 移动端布局

- 在现有 `max-width: 760px` 断点中继续使用相对定位，不改变左侧视频与右侧卡片的移动端媒体布局。
- `.experience-stage__copy` 的 `margin-top` 从 `8px` 增至 `24px`，实现适中下移。
- 大标题字号调整为 `clamp(32px, 10vw, 50px)`，保留两行排版并允许在 360px/390px 内容宽度内完整显示。
- 文字区与情绪切换器之间继续由现有弹性布局分配空间；若实测发生碰撞，只增加互动舞台最小高度，不改变右侧图片位置。

## 测试与验收

- 组件测试锁定左侧视频路径、可访问名称、无旧海报，以及右侧两张图片路径不变。
- 样式测试锁定桌面端 `bottom: 6%`、桌面标题流体字号、移动端 `margin-top: 24px` 和移动标题字号。
- 完整运行 Vitest 与 Vite production build。
- 浏览器在 1440×900、390×844、360×800 下验证：
  - 左侧视频使用 `/media/works/test-success.mp4`；
  - 右侧两张卡片的 bounding box 与修改前基线一致；
  - 标签、标题、中文说明的 bounding box 不互相重叠；
  - 文字区不超出互动舞台；
  - 页面无横向溢出。

## 非目标

- 不修改右侧杂志图片、卡片构图和情绪切换器。
- 不改变互动体验标题文案或中文说明内容。
- 不新增依赖、不联网、不生成新素材。
- 不修改 Hero、角色介绍、作品案例或联系方式板块。
