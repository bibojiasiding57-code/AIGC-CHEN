# AIGC-CHEN 角色介绍六卡精修设计

## 目标

在不改变现有角色介绍横向卡片轨道、滚动方式和三层文字结构的前提下，更新全部六张角色卡的数据，并将用户提供的黑色长裙与白色短裙图片分别绑定到第 5、6 张卡片。六张卡片在桌面端与 360px/390px 移动端保持统一尺寸、统一图片区域和一致的人物视觉密度。

## 保留范围

- 保留现有 `character-showcase`、`character-track` 与 `character-card` 组件结构。
- 保留横向滚动、滚动吸附、搜索和前后切换交互。
- 保留三层文字顺序：上方全大写小字 `discipline`、中间 Mixed Case 大字 `name`、底部中文 `title`。
- 保留无彩色色块的白色卡片视觉与现有悬停轻微放大效果。

## 六卡数据顺序

数据数组与渲染顺序严格一致，不进行异步排序：

1. `YOUTH CULTURE / POP AESTHETICS` / `Neo-Youth Icon` / `新世代青春图鉴`
2. `CINEMATIC FANTASY / GAME IP` / `Valkyrie Ascendant` / `史诗幻想女主角`
3. `OUTDOOR LIFESTYLE / AUTOMOTIVE` / `Desert Maverick` / `旷野风尚探索者`
4. `BEAUTY & LUXURY / HIGH FASHION` / `Ethereal Muse` / `高定纯净美学`
5. `HIGH LUXURY / PRESTIGE BRAND` / `Midnight Elegance` / `黑金奢华美学形象`
6. `EDITORIAL / MODERN FASHION` / `Urban Vanguard` / `都会摩登风尚图鉴`

前四张沿用现有图片。第 5 张使用 `generated_3afc6919-3d47-45ee-9075-b39eaa25c39b_mw0-1kw-1eja3 (1).png`（黑色长裙）；第 6 张使用 `generated_8ab4c916-04a4-412f-8fe7-9f123763c016_mw0-1kw-1dufl.png`（白色短裙）。两张图片复制到 `public/media/characters/`，使用稳定、可读的英文文件名。

## 图片布局与裁切

- 六张卡继续共用同一高度、圆角、图片容器和 `object-fit: cover`。
- 使用数据项中的 `position` 为各人物逐卡设置 `object-position`，不引入不同尺寸的图片容器。
- 第 5、6 张以与前四张相同的主体视觉密度为准，不追求完整全身展示；裁切重点为脸部、肩部与服装主体。
- 浏览器视觉验收时比较六张人物头部高度、主体占比与卡片底部文字区对齐；必要时只微调第 5、6 张的 `position`。

## 文本与响应式

- 正文区保持固定的三层垂直层级和现有间距。
- 长英文分类允许在有限宽度内自然换行，但不能与角色名重叠。
- 英文角色名使用流体字号，并在移动端保持清晰且不横向溢出。
- 中文说明保持单独一层，不截断、不覆盖。
- 360px/390px 下仍使用单卡横向滑动布局，卡片宽高与图片比例一致。

## 数据流与可访问性

- `characters` 数组是唯一数据源；React `key` 继续使用稳定的角色 `id`。
- 图片 `alt` 继续由角色名和中文说明组合生成。
- 搜索索引继续包含英文角色名、中文说明和英文分类。
- 图片加载失败沿用浏览器默认行为，不新增远程回退或联网依赖。

## 测试与验收

- 数据测试校验六卡数量、顺序、三层文本以及第 5、6 张本地图片路径。
- 组件测试校验六张卡均渲染且文本结构顺序不变。
- 样式测试校验统一图片容器、`object-fit: cover`、一致卡片尺寸和移动端规则。
- 完整运行 Vitest 与 Vite production build。
- 在桌面端及 390×844、360×800 浏览器尺寸下检查：无页面横向溢出、六卡顺序正确、图片容器一致、文字无重叠，第 5、6 张人物密度与前四张协调。

## 非目标

- 不重做角色介绍板块的工具栏、搜索框、按钮或整体背景。
- 不改变横向滚动为六列同时压缩展示。
- 不为新素材生成额外 AI 图片或添加彩色色块。
- 不修改作品案例、互动体验、Hero 或联系方式板块。
